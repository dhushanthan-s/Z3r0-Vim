## System prerequisites

Before running [`install.sh`](install.sh), ensure these are available (or use `./install.sh --install-deps` on Arch, Debian/Ubuntu, Fedora, or macOS with Homebrew to install the recommended set).

| Tier | Packages / tools | Role |
|------|------------------|------|
| **Required** | `git`, `nvim` | Lazy bootstrap, this script, Gitsigns |
| **Recommended** | `rg` (ripgrep), `fd`, `gcc` or `clang`, `make`, `curl`, `unzip` | Telescope search (`live_grep`, `find_files`), Treesitter parser builds, Mason downloads |
| **Optional** | `lazygit` | LazyGit.nvim (`<leader>gg`) |
| **As needed** | JDK, Rust, `gdb`/`lldb`, formatters via Mason, Nerd Fonts | Java/Rust/DAP/formatting; icons and media previews are not installed by `install.sh` |

`install.sh` checks for required tools and warns about missing recommended ones; it exits with an error only if `git` or `nvim` is missing.

## Comments

Comment.nvim uses its default keybinds (no overrides in config):

- **Line comment:** `gcc` (toggle), `[count]gcc`
- **Block comment:** `gbc` (toggle), `[count]gbc`
- **Visual:** `gc` (toggle line/block depending on selection)

## Alpha dashboard

When the dashboard is shown, the on-screen buttons use single keys (e.g. `f` find file, `e` new file, `p` find project, `r` recent files, `t` find text, `c` config, `q` quit). These are not global keybinds; they only work when the dashboard is focused.
